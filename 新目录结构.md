# 工程化目录结构设计

## 新目录结构

```
dailyNews/
├── src/                      # 源代码目录
│   ├── assets/              # 静态资源
│   │   ├── css/            # 样式文件
│   │   │   ├── common.css  # 公共样式
│   │   │   ├── home.css    # 首页样式
│   │   │   ├── archive.css # 归档页样式
│   │   │   └── detail.css  # 详情页样式
│   │   ├── js/             # JavaScript 文件
│   │   │   ├── utils.js    # 工具函数
│   │   │   ├── home.js     # 首页逻辑
│   │   │   ├── archive.js  # 归档页逻辑
│   │   │   └── detail.js   # 详情页逻辑
│   │   └── lib/            # 第三方库
│   │       ├── showdown.min.js
│   │       └── github-markdown.min.css
│   ├── pages/              # 页面文件
│   │   ├── index.html      # 首页
│   │   ├── archive.html    # 归档页
│   │   └── detail.html     # 详情页
│   └── crawler/            # 爬虫相关
│       ├── index.js        # 爬虫主程序
│       └── fetch.js        # 请求封装
├── public/                  # 公共静态文件
│   └── news/               # 新闻数据
│       ├── catalogue.json  # 目录索引
│       └── *.md            # 文字稿文件
├── docs/                    # 文档
│   ├── 项目说明.md
│   └── 启动说明.md
├── .gitignore
├── package.json
└── README.md
```

## 改进点

1. **分离关注点** - HTML/CSS/JS 分离
2. **模块化** - 公共样式/工具函数复用
3. **清晰结构** - src 源码 / public 数据 / docs 文档
4. **易维护** - 文件职责单一,便于定位
5. **可扩展** - 便于添加新页面和功能
